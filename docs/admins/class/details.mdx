## Register Members

Admins can register Members to Classes manually.

#### Steps

1. Select the <Icon path={mdiAccountMultiplePlus} /> button near the top right.
2. A search bar appears in the [Attendance List][AttendanceList] section.
3. Search of Members by last name first.
4. Click <Button>Register Members</Button> button.
5. Confirm the action by typing the confirmation text.
6. See notifications about successfully registering members and about email notices sent.

:::tip
You can add multiple Members at once and they will be added in the order shown.
:::

:::info
Any Members registered over the [Seats](#seats) limit will be added to the [Waiting List][WaitingList]
:::

### Override Seats

:::tip
You can stop Members from being added to the [Waiting List][WaitingList] by selecting the **Override Max Seats** option just under where you search for members to register.
:::

## Unregister Members

To unregister Member from a Class please see the demonstration video below.

#### Steps

1. Select the <Icon path={mdiAccountMultipleMinus} /> button near the top right.
2. Attendance List Member will have a checkbox appear next to their name.
3. Select Member(s) you want to unregister.
4. Click <Button>Remove Select Members</Button> button.
5. Confirm the action by typing the confirmation text.
6. See notifications about successfully removing members and about email notices sent.

:::note
If you unregister Member(s) from a full Class with Members on the [Waiting List][WaitingList], the spots will be automatically filled by the [Waiting List][WaitingList] Members in a [FIFO](https://en.wikipedia.org/wiki/FIFO_and_LIFO_accounting#:~:text=%22FIFO%22%20stands%20for%20first%2D,is%20the%20cost%20expensed%20first.) basis.
:::

## Print Class Attendance List

Admins can print off Class Attendance List for each day of the Class.

#### Steps

1. Click <Button>Print Attendance List</Button> button.
2. Enter the number corresponding to the day you want to print off.
3. Then client <Button>Print</Button>

:::note
Enter the number for the corresponding Class day for which you wish to acquire the attendance list

## Copy Attending Member Emails

Admins can copy all the Attendance List Member emails.

#### Steps

1. Click <Icon path={mdiEmail} /> button.

3. Paste the email list where you need them.

:::tip
This is useful for sending a mass email. Simply follow the directions and `Ctrl+v`(paste) where you need the email, likely the **TO** field for an email.

## Complete Class

Completing the Class will automatically update all the Member course certificate records.

#### Steps

1. Click <Icon path={mdiCalendarCheck} /> button.
2. Select the Member who **did not** complete the Class.
3. Click the <Button>Complete Class</Button>
4. Enter the confirmation text, and click <Button variant="filled">Complete</Button>
5. See notification about successfully completing the Class and email notices to Members sent.

:::note
Each Member which successfully completed the Class will receive an email notice once the training coordinator completes the class.
:::

## Waiting List

The Waiting List is a place to add Members once the [Attendance List][AttendanceList] has been filled.

If the [Attendance List][AttendanceList] is full and there are member(s) on the Waiting list, any time a Member is [Unregistered][AdminUnregister] by an admin or they [unregister themselves][MemberUnregister], the first Member(s) on the Waiting list will be transferred to the [Attendance List][AttendanceList] and sent and email notification letting them know they have been Register to the Class.

### Add Members to Waiting List

Members get added to the Waiting List automatically when the [Attendance List][AttendanceList] is full and either the admin [Registered][AdminRegister] them or they [register][MemberRegister] themselves.

Admins can force members onto the [Attendance List][AttendanceList] with the [Override Seats][OverrideSeats] option.

### Remove Member from Waiting List

Admins can manually remove Members from the Waiting List.

#### Steps

1. Select the Member(s) on the Waiting List to be removed.
3. Click the <Button>Remove Members</Button> button.
4. Enter the confirmation text, and click <Button variant="filled">Remove</Button>
5. See notification about successfully removing member(s).

:::note
No email notifications are sent with regards to the removal of Member from the Waiting List.
:::

## Update Class

### Change Class Dates

Admins can change the Dates associated with the Class.

You can:
- Add new days
- Remove days
- Change start time with automatic end time update
- Change start time **without** automatic end time update
- Reset Dates back to original

#### Steps

1. Click the <Icon path={mdiPencil} /> **Edit Class** Button
2. Make the changes you need to.
3. Select <Icon path={mdiCheckboxMarkedCircleOutline} /> **Update Class** button.
4. You can cancel your changes by selecting the <Icon path={mdiPencilOff} /> **Cancel Edit** button.
5. See notification about successfully updating the Class.

:::note
When you change the dates in anyway, an email notice will be sent to the Attendance List of Members.
Waiting List member will **not** receive an email.
:::

### Seats

Seats is how many members can be trained at once with each Class. Seats will default to the number of [Spots](../course/details#spot) set when [creating the Course](../course/create#page-2).
Admins can change the default value when [creating the Class](./create#page-2), and after the Class has been created

The Seats limit will determine when Members are added to the [Waiting List][WaitingList].

### Trainer

The Trainer field is optional when [creating a Class](./create#page-2). This is for your records, and a blank Trainer field will **not** prevent an admin from [Completing a Class](#complete-the-class).

### Registration Closed

Registration Closed, when set will prevent member from [registering themselves](../../member/class-calendar#register) to a Class. However, admins are still able to [register member](#register-members).

## Delete Class

#### Steps

1. Select the <Icon path={mdiDelete} /> button.
2. Confirm the delete action by typing the confirmation text.
3. See notifications about successfully deleting the Class.

:::warning
Deleting a Class should only be done for Classes that have not yet happened.
:::

:::note
Classes cannot be deleted until all members have been [Unregistered][AdminUnregister] and removed from the [Waiting List][WaitingList].

[Completed][ClassComplete] Classes cannot be deleted
:::
